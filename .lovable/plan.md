

# Performance Audit Implementation Plan

## Overview
This plan implements the findings from the CogibleND Performance Audit v1.0 document. All changes are **pure refactors** — zero changes to routes, permissions, business rules, database, APIs, UX flows, or navigation. We follow the document's recommended execution sequence (safest first).

---

## Phase 1: Safe Deletions (Minimal Risk)

### 1.1 Remove 6 Unused UI Components (PERF-H01)
Confirmed zero imports across the codebase for all six:
- `src/components/ui/hover-card.tsx`
- `src/components/ui/menubar.tsx`
- `src/components/ui/navigation-menu.tsx`
- `src/components/ui/context-menu.tsx`
- `src/components/ui/resizable.tsx`
- `src/components/ui/aspect-ratio.tsx`

Also remove `carousel.tsx` and `drawer.tsx` (also zero imports confirmed). Remove corresponding Radix packages from `package.json`: `@radix-ui/react-hover-card`, `@radix-ui/react-menubar`, `@radix-ui/react-navigation-menu`, `@radix-ui/react-context-menu`, `@radix-ui/react-aspect-ratio`.

### 1.2 Move Test Deps to devDependencies (PERF-C06)
Move `happy-dom` and `@testing-library/user-event` from `dependencies` to `devDependencies` in `package.json`.

### 1.3 Remove next-themes (PERF-M03)
In `src/components/ui/sonner.tsx`: remove `useTheme` import, use static `theme = "light"`. Remove `next-themes` from `package.json`.

## Phase 2: Toast Consolidation (Low Risk)

### 2.1 Consolidate to Sonner Only (PERF-C05)
Verified: `useToast` from shadcn is imported by **zero** feature components (only by its own files: `toaster.tsx` and `use-toast.ts`). Safe to remove:
- Remove `<Toaster />` (shadcn) from `App.tsx` line 281, keep `<Sonner />`
- Delete `src/components/ui/toaster.tsx`, `src/components/ui/use-toast.ts`, `src/hooks/use-toast.ts`, `src/components/ui/toast.tsx`
- Remove `@radix-ui/react-toast` from `package.json`

---

## Phase 3: Bundle Optimization (Low Risk)

### 3.1 Lazy-Load 16 Enrollment Pages (PERF-C02)
Replace the barrel import in `App.tsx` lines 56-73 with individual `lazy()` imports:
```
const EnrollRegistration = lazy(() => import('@/pages/enroll/Registration'));
const EnrollParticipationMode = lazy(() => import('@/pages/enroll/ParticipationMode'));
// ... all 16 pages
```
Wrap enrollment routes with `<LazyRoute>` (already exists as helper). The barrel file `src/pages/enroll/index.ts` can remain for backward compat but will no longer be imported in App.tsx.

### 3.2 Dynamic XLSX Imports (PERF-H05)
Convert static `import * as XLSX from "xlsx"` to dynamic `const XLSX = await import("xlsx")` at point-of-use in:
- `QuestionBankPage.tsx`
- `QuestionImportDialogOptimized.tsx`
- `ProficiencyExcelExport.ts`
- `InterviewKitExcelExport.ts`

Make containing functions `async` where needed. No change to XLSX operations.

### 3.3 Add Supabase Vendor Chunk (PERF-H06)
Add `'vendor-supabase': ['@supabase/supabase-js']` to `vite.config.ts` manualChunks.

---

## Phase 4: Architecture Fixes (Medium Risk)

### 4.1 Scope EnrollmentProvider (PERF-C01)
Move `<EnrollmentProvider>` in `App.tsx` from wrapping ALL routes to wrapping only:
- `/enroll/*` routes
- `/dashboard` route (provider dashboard)

Keep `<AuthProvider>` global. No route paths, guards, or components change.

### 4.2 Add Module-Level Error Boundaries (PERF-M02)
Wrap each route group in `App.tsx` with `<ErrorBoundary>`:
- `/admin/*` → `componentName="Admin Portal"`
- `/enroll/*` → `componentName="Enrollment"`
- `/reviewer/*` → `componentName="Reviewer Portal"`
- `/org/*` → `componentName="Organization Portal"`
- `/pulse/*` → `componentName="Pulse"`

Keep outer App-level boundary as catch-all.

---

## Phase 5: Component Optimization (Minimal Risk)

### 5.1 React.memo on High-Frequency Components (PERF-H08)
Wrap with `React.memo`:
- `StatusBadge.tsx`
- `DisplayOrderCell.tsx`
- `MasterDataActions.tsx`
- `DeleteConfirmDialog.tsx`

No internal logic changes.

### 5.2 AdminSidebar ARIA + Handler Extraction (PERF-H04)
- Add `aria-label="Admin Navigation"` to root nav
- Add `aria-current="page"` to active menu item
- Extract inline `onClick` handlers into a memoized `handleNavigate` via `useCallback`
- No changes to menu items, order, icons, paths, or permission guards
